# 输出格式统一规范

> 强制表格输出统一规范 —— 郭帆导演工作室所有输出文件必须遵循

---

## 1. 文件头元数据格式

### 标准元数据块

所有输出文件必须以标准 YAML 元数据块开头，格式如下：

```
---
项目名称:       《项目名称》
集数:           第 N 集
文件类型:       [剧本/分镜/音频/封面]
版本:           V1.0
创建日期:       YYYY-MM-DD
创建人:         
---
```

### 元数据字段规则

| 字段 | 必填 | 格式要求 |
|------|------|----------|
| `项目名称` | 是 | 中文书名号《》，前后对齐缩进 |
| `集数` | 是 | "第 N 集"格式 |
| `文件类型` | 是 | 使用方括号枚举值 |
| `版本` | 是 | "V" + 数字 + "." + 数字 |
| `创建日期` | 是 | YYYY-MM-DD |
| `创建人` | 否 | 中文姓名 |

### 元数据排版规则

- 冒号为**全角冒号** `：`
- 内容与冒号之间占两格（对齐美观）
- 字段前方无缩进
- 元数据块使用代码块标记 `---` 包裹

## 2. Markdown 格式规范

### 2.1 标题层级规范

| 层级 | 标记 | 用法 | 编号规则 |
|------|------|------|----------|
| 一级标题 | `# ` | 文档标题（仅一个） | 不编号 |
| 二级标题 | `## ` | 主要章节 | 数字编号（如 1./2./3.） |
| 三级标题 | `### ` | 子章节 | 数字编号（如 1.1/1.2/1.3） |
| 四级标题 | `#### ` | 段落标题 | 少量使用，不编号 |

**规则：**
- 一级标题全文仅一个，即文件主标题
- 二级标题后必须空一行再继续
- 标题与内容之间空一行
- 不在标题末尾添加标点符号

### 2.2 表格对齐规则

| 对齐方式 | 标记 | 使用场景 |
|----------|------|----------|
| 左对齐 | `:---` | 文本内容、描述列、备注列 |
| 居中对齐 | `:---:` | 字段名称、编号列、编码列 |
| 右对齐 | `---:` | 数字列、时长列、百分比列 |

**示例：**

```
| 左对齐文本 | 居中对齐字段 | 右对齐数字 |
|:-----------|:------------:|-----------:|
| 内容描述   |    TX-001    |        3s |
| 详细描述   |    JT-002    |        5s |
```

**强制规则：**
- 表格必须有表头分隔行（不能省略）
- 表头行使用**加粗**格式
- 表格列数必须一致，每行单元格数相同
- 表格前后各空一行
- 表格内单元格内容 ≤ 30 字（超过应换行或拆分）

### 2.3 表头命名规则

| 命名规范 | 说明 | 示例 |
|----------|------|------|
| 使用中文 | 所有表头使用中文，不中英混杂 | ❌ "Scene No." → ✓ "场号" |
| 字数 ≤ 6 字 | 表头简洁明了 | ❌ "本镜头持续时长" → ✓ "时长" |
| 名词为主 | 使用名词性短语 | "景别"、"运镜"、"画面描述" |
| 避免标点 | 表头中不使用标点符号 | ❌ "备注？" → ✓ "备注" |
| 统一字体 | 表头加粗 | `**镜头号**` |

### 2.4 色彩标注规则

#### 标注格式

```
色彩标注统一使用十六进制色码
格式：#RRGGBB（大写）
```

**规则：**
- 颜色代码统一大写十六进制（#FFB6C1 而非 #ffb6c1）
- 颜色代码后加中文颜色名称（可选）：`#FFB6C1（浅粉）`
- 同一文件中相同颜色使用相同色码值
- 色码与文字之间空一格

#### 色调标注在分镜中的引用格式

```
色调：[模板编号] - [模板名称]
色调：A - 冷峻都市
色调：B - 暖甜恋语
色调：C - 复古胶片
```

### 2.5 代码块与引用规则

| 元素 | 标记 | 使用场景 |
|------|------|----------|
| 代码块 | ``` ``` | 示例剧本、示例分镜、YAML 配置 |
| 行内代码 | `` ` ` `` | 文件名、参数名、编码值 |
| 引用块 | `> ` | 文档说明、注意事项、免责声明 |
| 分割线 | `---` | 章节间分隔（宽分隔） |

**缩进规则：**
- 代码块内容不缩进（从列 1 开始）
- 引用块嵌套：`> ` → `>> ` → `>>> `

## 3. 缩进与层级规范

### 3.1 列表缩进规范

```
一级列表（无序）：- 内容
  二级列表（无序）：  - 内容（缩进 2 格）
    三级列表（无序）：  - 内容（缩进 4 格）

一级列表（有序）：1. 内容
  二级列表（有序）：  a. 内容（缩进 2 格）
    三级列表（有序）：  i. 内容（缩进 4 格）
```

### 3.2 内容缩进规则

| 内容类型 | 缩进规则 |
|----------|----------|
| 章节标题 | 无缩进 |
| 正文段落 | 无缩进，段间空一行 |
| 列表项 | 一级无缩进，二级缩进 2 空格 |
| 代码块 | 无缩进块（保持代码块标记与内容对齐） |
| 表格 | 无缩进 |
| 引用 | 引用标记 `>` 无缩进，内容紧随 |

### 3.3 空白行规则

- 章节之间（两个二级标题之间）：空两行
- 标题与内容之间：空一行
- 表格前后：各空一行
- 代码块前后：各空一行
- 列表项之间：同一级列表不空行，不同级列表之间空一行

## 4. 标记与标注规范

### 4.1 剧本标记

| 标记 | 含义 | 使用范围 |
|------|------|----------|
| `【动作】` | 动作描写 | 行为描述，每个动作 ≤ 30 字 |
| `【节奏：】` | 节奏标注 | 加速/减速/骤停/反转/悬念 |
| `【音效：】` | 音效标注 | AMB/SFX/EMO/TRN/IMP |
| `【配乐：】` | 配乐标注 | 风格/情绪/BPM/起止/音量 |
| `（OS）` | 画外音 | 画外音人物标识 |

### 4.2 分镜标记

| 标记 | 含义 | 使用范围 |
|------|------|----------|
| `【色调：】` | 色调绑定 | A/B/C 三套模板 |
| `（无台词）` | 无台词镜头 | 纯画面镜头 |
| `（OS）` | 画外音 | 画外音台词 |

### 4.3 强调标记

- 关键数值使用**加粗**（如**≥ 60%**）
- 文件名、参数名使用 `` `行内代码` ``（如 `global_param_config.yaml`）
- 注释放置在括号内（如 `（仅限特写镜头）`）
- 禁止使用下划线 `_` 表示强调

## 5. 资产引用路径规范

### 5.1 内部引用

```
# 引用同一项目下的其他文件
参见：`prompt_template/文本剧本模板.md`
参见：`prompt_template/视频分镜模板.md`
```

### 5.2 同级文件引用

```
# 引用同目录下的其他文件
详情见：`../output_format_rule.md`
```

### 5.3 资产路径规则

| 资产类型 | 路径格式 | 示例 |
|----------|----------|------|
| 模板文件 | `prompt_template/文件名.md` | `prompt_template/文本剧本模板.md` |
| 配置文件 | `文件名.yaml` | `global_param_config.yaml` |
| 图片资产 | `assets/图片名.png` | `assets/cover_example.png` |
| 音效资产 | `assets/audio/音效名.wav` | `assets/audio/heartbeat.wav` |

**规则：**
- 路径分隔符统一使用正斜杠 `/`
- 路径名使用英文小写，文件名包含中文用中文
- 所有资产引用在文档中使用行内代码 `` ` ` `` 包裹

## 6. 文件命名规则

### 6.1 命名格式

```
[类型前缀]_[描述名].[扩展名]
```

### 6.2 命名规范

| 文件类型 | 命名格式 | 示例 |
|----------|----------|------|
| 剧本 | `剧本_项目名_集数.md` | `剧本_替身前妻她不哭了_01.md` |
| 分镜 | `分镜_项目名_集数.md` | `分镜_替身前妻她不哭了_01.md` |
| 音频配置 | `音频_项目名_集数.md` | `音频_替身前妻她不哭了_01.md` |
| 封面配置 | `封面_项目名_集数.md` | `封面_替身前妻她不哭了_01.md` |
| 全局配置 | `global_param_config.yaml` | 固定文件名 |
| 输出规范 | `output_format_rule.md` | 固定文件名 |

## 7. 版本与更新记录

### 7.1 版本号规则

| 版本号 | 含义 | 触发条件 |
|--------|------|----------|
| V1.0 | 初始版本 | 首次交付 |
| V1.x | 修订版 | 内容修改、格式调整 |
| V2.0 | 重大更新 | 结构变更、标准重订 |

### 7.2 更新记录格式

每个文件末尾可选附加更新记录：

```
---

## 更新记录

| 版本 | 日期 | 修改人 | 修改内容 |
|------|------|--------|----------|
| V1.0 | YYYY-MM-DD | — | 初始创建 |
| V1.1 | YYYY-MM-DD | — | 修订内容说明 |
```
